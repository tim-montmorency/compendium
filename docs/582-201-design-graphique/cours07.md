## UX

<!-- https://www.figma.com/resource-library/what-is-ux-design/ -->

L'expérience utilisateur (UX) est la qualité de l’expérience vécue par une personne quand elle utilise une interface.

Une bonne UX, c’est quand l’utilisateur :
	•	comprend où il est
	•	sait quoi faire
	•	réussit vite
	•	fait peu d’erreurs
	•	se sent en confiance

!!! info "Différence entre UX et UI"

    UI (User Interface) = l’apparence et les composants visuels (boutons, menus, couleurs, typographies).

    UX = le parcours, la logique, la clarté, l’effort mental, le “flow”.


## Parcours utilisateur

> Qui fait quoi, dans quel ordre ?

Un parcours (user journey / user flow), c’est la suite d’étapes qu’un utilisateur suit pour atteindre un objectif.

2.1 Définir l’objectif (la “job to be done”)

Exemples d’objectifs :
	•	acheter un billet
	•	réserver une table
	•	trouver un horaire
	•	s’inscrire à un cours
	•	télécharger un document

2.2 Flow minimal (à viser)

Un bon parcours est souvent :
	•	court
	•	prévisible
	•	sans surprise
	•	sans “cul-de-sac” (pas de page qui bloque)

2.3 Points de friction (à repérer)
	•	trop d’étapes / trop de champs
	•	mots vagues (“Soumettre”, “Continuer”, “Autre”)
	•	informations essentielles cachées
	•	erreurs difficiles à corriger
	•	manque de feedback (on ne sait pas si ça a marché)

## Hiérarchie visuelle

guider l’œil et les priorités

La hiérarchie, c’est l’ordre dans lequel on veut que l’utilisateur voie et comprenne.

3.1 Les leviers de hiérarchie (les plus courants)
	•	Taille (plus grand = plus important)
	•	Contraste (valeur/couleur/gras)
	•	Position (haut, gauche, centre, zones dominantes)
	•	Espace (marges, respiration, séparation)
	•	Répétition / cohérence (styles stables)
	•	Proximité (ce qui est proche “va ensemble”)
	•	Typographie (niveau de titres, interlignage, longueur de ligne)
	•	Images (une image attire souvent avant le texte)

3.2 Règle simple

Si tout crie, rien n’est important.
On veut 1 priorité claire, puis des niveaux secondaires.

3.3 Test éclair (5 secondes)

En 5 secondes, un utilisateur devrait pouvoir répondre :
	•	C’est quoi cette page?
	•	Pour qui?
	•	Qu’est-ce qu’on attend de moi (CTA)?

## Patterns de lecture : Z et F

Ces patterns décrivent comment l’œil scanne une interface (surtout sur écran).

4.1 Pattern en Z (pages simples / visuels forts)

Souvent quand :
	•	peu de texte
	•	une structure très claire
	•	un “hero” avec un CTA

Trajet typique :
	1.	haut-gauche → haut-droite
	2.	diagonale vers le bas-gauche
	3.	bas-gauche → bas-droite

Implication design
	•	place les infos clés sur le trajet (logo, promesse, CTA)
	•	attention aux éléments “hors trajet” : ils seront moins vus

4.2 Pattern en F (pages riches en texte / listes / résultats)

Souvent quand :
	•	beaucoup de contenu
	•	articles, listes, pages de recherche
	•	sections répétitives

Trajet typique :
	•	une barre horizontale en haut
	•	une deuxième barre horizontale plus bas (plus courte)
	•	puis une lecture verticale à gauche

Implication design
	•	titres, sous-titres, mots-clés à gauche
	•	paragraphes courts, listes, intertitres
	•	“scannabilité” avant “littérature”

## Heuristiques (Nielsen) : vérifier la qualité UX

Une heuristique = une règle de bon sens testée en UX pour détecter des problèmes rapidement.
On s’en sert pour faire une évaluation heuristique d’une interface.

Les 10 heuristiques (version pratique)
	1.	Visibilité de l’état du système : on sait ce qui se passe (chargement, succès, erreur).
	2.	Correspondance avec le monde réel : mots et concepts simples, familiers.
	3.	Contrôle et liberté : annuler, revenir en arrière, corriger facilement.
	4.	Cohérence et standards : mêmes patterns, mêmes termes, comportements attendus.
	5.	Prévention des erreurs : éviter de piéger l’utilisateur.
	6.	Reconnaissance plutôt que mémorisation : options visibles, pas besoin de se souvenir.
	7.	Flexibilité et efficacité : raccourcis, rapidité pour utilisateurs fréquents.
	8.	Design minimaliste : enlever le bruit, garder l’essentiel.
	9.	Aider à reconnaître et corriger les erreurs : message clair + solution.
	10.	Aide et documentation : au besoin, facile à trouver, courte, utile.
