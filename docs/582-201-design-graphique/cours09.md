Le bon réflexe : 1) structure → 2) contenu → 3) style → 4) itérations → 5) tests.

Voici des prompts adaptés à ce workflow Make (courts, actionnables).

A) Prompt “structure d’abord”

Crée une landing page très simple (wireframe gris), avec : Hero (H1 + sous-titre + CTA), 3 bénéfices, preuve sociale, section “comment ça marche”, FAQ, footer. Priorité : hiérarchie + spacing, pas le style.

B) Prompt “contenu réaliste”

Remplace le texte générique par du vrai microcopy en français pour une audience de cégep (18–25). Donne 3 variantes de CTA orientées bénéfice. Garde les phrases courtes.

C) Prompt “style guidé par contraintes”

Applique un style sobre “SaaS” : 1 couleur d’accent max, typographie lisible, boutons cohérents, cartes avec ombres légères. N’ajoute pas de sections.

D) Prompt “itération sans régénérer”

Fais une critique : 5 problèmes de clarté/CTA/hiérarchie. Puis corrige uniquement : titres, espacements, alignements, tailles de texte. Interdit de changer la structure.

E) Prompt “référence obligatoire” (le bouton +)

Je joins un design de référence. Reproduis la structure et les proportions (grille, marges, densité), mais change la marque (nom, texte, images). Conserve l’esprit, pas un clone.

F) Prompt “A/B test en classe”

Génère 2 variantes du Hero (A et B) avec le même contenu. Pour chacune : hypothèse + ce qu’on mesure + 2 questions de test “5 secondes”. -->
